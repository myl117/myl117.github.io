Using gRPC for Microservice Communication

12 Jan, 2026

In traditional microservice architectures, communication between services is done via REST or sometimes for event driven communication, protocols like MQTT are employed. In a RESTful architecture, resources are identified by URIs (Uniform Resource Identifiers) and operations on those resources are performed using standard HTTP methods. Data Transfer Objects (DTOs) are represented as JSON or XML which are both lightweight and human readable formats.

gRPC on the other hand is a relatively newer framework which facilitates efficient communication using RPCs (Remote Procedure Calls) enabling services to call functions on other machines as if they were local. It is built on the use of a mechanism called Protobufs (Protocol Buffers) which supports strongly typed service contracts for DTOs and binary data encoding as opposed to JSON. Protobuf contracts are defined in .proto files which are used to generate code that can be used by a microservice akin to a library or SDK.

Data encoded in a binary form is up to 5 times more space efficient and faster to serialise and deserialise than JSON/XML with the obvious caveat of not being human readable. Unless you are a living Mentat of course ;)

Building a Simple Library System

Let’s create a simple library system that will demonstrate a simple microservices architecture (Node.js client and Java server) with gRPC communication and touch upon the fundamentals of the RPC protocol. It will allow us to view a list of books or get information about a specific one.

Let’s first create a Protobuf contract that will define the methods for our service.

 1syntax = "proto3";
 2
 3option java_package = "com.myl117.java.library.grpc.service";
 4option java_outer_classname = "LibraryProto";
 5
 6service LibraryService {
 7  // Get information about a book by ID
 8  rpc GetBook (BookRequest) returns (BookResponse);
 9
10  // List all books
11  rpc ListBooks (Empty) returns (BookList);
12}
13
14message Empty {}
15
16message BookRequest {
17  int32 id = 1;
18}
19
20message BookResponse {
21  int32 id = 1;
22  string title = 2;
23  string author = 3;
24  bool available = 4;
25}
26
27message BookList {
28  repeated BookResponse books = 1;
29}

Now that we have all the dependencies for our project setup, we can now focus on the gRPC server implementation. It is important this service sits in a new package, the same one defined in your proto file. Before writing the service code, compile your Maven project to generate code from your proto file.

 1package com.myl117.java.library.grpc.service.java_grpc_library_service.service;
 2
 3import com.myl117.java.library.grpc.service.LibraryProto.BookRequest;
 4import com.myl117.java.library.grpc.service.LibraryProto.BookResponse;
 5import com.myl117.java.library.grpc.service.LibraryProto.BookList;
 6import com.myl117.java.library.grpc.service.LibraryServiceGrpc;
 7import io.grpc.Server;
 8import io.grpc.ServerBuilder;
 9import io.grpc.stub.StreamObserver;
10import jakarta.annotation.PostConstruct;
11import jakarta.annotation.PreDestroy;
12import org.springframework.stereotype.Component;
13
14import java.io.IOException;
15import java.util.ArrayList;
16import java.util.List;
17import java.util.Optional;
18
19@Component
20public class GrpcServer {
21
22  private Server server;
23
24  // In-memory list of books
25  private final List <BookResponse> books = new ArrayList <>();
26
27  @PostConstruct
28  public void start() throws IOException {
29    // Add sample books
30    books.add(BookResponse.newBuilder()
31      .setId(1)
32      .setTitle("1984")
33      .setAuthor("George Orwell")
34      .setAvailable(true)
35      .build());
36    books.add(BookResponse.newBuilder()
37      .setId(2)
38      .setTitle("The Hobbit")
39      .setAuthor("J.R.R. Tolkien")
40      .setAvailable(false)
41      .build());
42      books.add(BookResponse.newBuilder()
43      .setId(3)
44      .setTitle("Sunrise on the Reaping")
45      .setAuthor("Suzanne Collins")
46      .setAvailable(false)
47      .build());
48
49    // Start gRPC server on port 9090
50    server = ServerBuilder.forPort(9090)
51      .addService(new LibraryServiceImpl())
52      .build()
53      .start();
54
55    System.out.println("gRPC server started on port 9090");
56  }
57
58  @PreDestroy
59  public void stop() {
60    if (server != null) {
61      server.shutdown();
62      System.out.println("gRPC server stopped");
63    }
64  }
65
66  // Implementation of the gRPC service
67  private class LibraryServiceImpl extends LibraryServiceGrpc.LibraryServiceImplBase {
68
69    @Override
70    public void getBook(BookRequest request, StreamObserver <BookResponse> responseObserver) {
71      Optional <BookResponse> book = books.stream()
72        .filter(b -> b.getId() == request.getId())
73        .findFirst();
74
75      responseObserver.onNext(book.orElse(
76        BookResponse.newBuilder()
77        .setId(0)
78        .setTitle("Not Found")
79        .setAuthor("")
80        .setAvailable(false)
81        .build()
82      ));
83      responseObserver.onCompleted();
84    }
85
86    @Override
87    public void listBooks(com.myl117.java.library.grpc.service.LibraryProto.Empty Empty, StreamObserver <BookList> responseObserver) {
88      BookList.Builder builder = BookList.newBuilder();
89      builder.addAllBooks(books);
90      responseObserver.onNext(builder.build());
91      responseObserver.onCompleted();
92    }
93  }
94}

As you can see in the code, we import methods generated by the protocol buffer compiler. The code itself is fairly straightforward to understand but there are more powerful constructs we can work with as well as features like streaming and deadlines. But I’ll let you, the reader, expand on this project and make those implementations.

We now have a gRPC server but one that is very lonely and would like a client to be able to communicate with it. So let’s create a client. Let’s create a new Typescript file project, install the necessary modules/types and copy the same proto file used in the java server.

1npm install @grpc/grpc-js @grpc/proto-loader
2npm install -D typescript ts-node @types/node

In our client.ts, we’ll load the proto definition and create a client. Now that the library is set up, we can make procedure calls on our Java gRPC server as if they were local procedures!

 1import * as grpc from "@grpc/grpc-js";
 2import * as protoLoader from "@grpc/proto-loader";
 3import path from "path";
 4
 5// Load proto definition
 6const packageDef = protoLoader.loadSync(
 7  path.join(__dirname, "/proto/library.proto"),
 8  {
 9    keepCase: true,
10    longs: String,
11    enums: String,
12    defaults: true,
13    oneofs: true,
14  }
15);
16
17// Convert proto into gRPC object
18const grpcObject = grpc.loadPackageDefinition(packageDef) as any;
19
20// Create client
21const client = new grpcObject.LibraryService(
22  "localhost:9090",
23  grpc.credentials.createInsecure()
24);
25
26client.getBook({ id: 1 }, (err: grpc.ServiceError | null, response: any) => {
27  if (err) {
28    console.error(err.message);
29    return;
30  }
31
32  console.log("Book:", response);
33});
34
35client.listBooks({}, (err: grpc.ServiceError | null, response: any) => {
36  if (err) {
37    console.error(err.message);
38    return;
39  }
40
41  console.log("Books:", response.books);
42});

And there you have it! A fully working client-server demo which if you’ve followed all the steps correctly, should display the books we defined on our server in the terminal. Although we’ve covered the fundamentals of gRPC, this is only the tip of the iceberg and I would like to encourage you to further explore more advanced concepts such as streaming, authentication, deadlines and more.
